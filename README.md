# StumbleClone

3D party-game prototype (Stumble Guys-style) — player vs 7 bots across 3 level modes.

## Stack
- **Unity 6000.4.8f1** + **URP**
- New Input System
- Unity AI Navigation (NavMesh)
- Cinemachine (3rd-person follow)

## Getting started
1. Open Unity Hub → Add → select this folder (`unity-projects/StumbleClone/`).
2. First open will resolve packages and import URP — let it finish (this takes a few minutes the first time).
3. Edit > Project Settings > **Tags & Layers** — add the following before pressing Play:
   - Tags: `Player`, `Bot`, `Finish`, `Killzone`, `PushPad`, `Respawn`
   - User Layer 8: `Player`, 9: `Bot`, 10: `Obstacle`, 11: `Ground`, 12: `Killzone`
4. Open `Assets/Scenes/MainMenu.unity` and press Play.
5. To import a real character, see `Assets/IMPORT_GUIDE.md` (created by the review pass).

```
  Models/        # Drop Mixamo FBX here

  Settings/      # URP asset, render-pipeline settings

```

## Architecture contract
All gameplay scripts target the contract in `Assets/Scripts/Core/`:

- `IRacer` — implemented by `PlayerController` and `BotController`. Defines `Knockback`, `Eliminate`, `Finish`, `Respawn`, and the `OnFinished`/`OnEliminated` events.
- `GameEvents` — global event bus (LevelStarted, LevelEnded, RacerFinished, RacerEliminated, etc.).
- `RacerRegistry` — lightweight static list of all `IRacer` instances; racers self-register in `OnEnable`.
- `GameConstants` — layer IDs, tags, default tuning knobs.

This contract is owned by the Core assembly. Other scripts reference it but should not modify Core unless the contract genuinely needs to change.
